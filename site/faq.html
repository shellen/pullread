Getting Started

What is Pull Read?

Pull Read is a macOS menu bar app for reading RSS feeds, bookmarked articles, and ebooks. It saves everything as local Markdown files, so your reading library is always yours — searchable, portable, and never locked into a platform.

How do I install it?

Download the DMG from the releases page, drag Pull Read to your Applications folder, and launch it. It lives in your menu bar — click the icon to open the viewer or access settings.

Where are my articles stored?

Articles are saved as Markdown files in ~/Documents/PullRead by default, organized in date folders (2024/02/article-title.md). You can change this path in Settings. Since they're plain Markdown with YAML frontmatter, you can read them with any text editor, Obsidian, or other Markdown tools.

Does it work offline?

Yes. Once articles are synced, everything works offline — reading, highlighting, notes, search, and text-to-speech (with Apple's built-in voices). AI features and feed syncing need an internet connection.

Feeds & Content

What kinds of feeds can I subscribe to?

Pull Read supports RSS, Atom, RDF, and JSON Feed formats. It also has smart URL resolution — paste a YouTube channel URL, a Reddit subreddit, or a blog homepage, and Pull Read will find the feed for you.

Can I import feeds from another reader?

Yes. Pull Read supports OPML import. Export your subscriptions from your current reader as an OPML file, then import it in Settings > Feeds.

Can I save individual articles without subscribing to a feed?

Yes, several ways:

  • Quick Add — Click the + button in the sidebar and paste a URL
  • Bookmarklet — Drag the bookmarklet from the setup tour to your browser's bookmarks bar, then click it on any page
  • Drop a file — Drag a .md file into the Pull Read window
Does it support podcasts?

Yes. Any RSS feed with audio enclosures shows up as a podcast with an inline player. You'll see episode descriptions and can play audio directly in the reader.

Can I read ebooks?

Yes. Drop an EPUB file into your Pull Read folder and it appears in your library with chapter navigation, table of contents, and cover images. Search for is:book to filter your book collection.

Reading & Highlights

How do I highlight text?

Select any text in an article and a highlight toolbar appears. Choose a color (yellow, green, blue, or pink) and optionally add a note. Highlights are saved automatically and shown as footnotes in the margin.

What's the difference between highlights, notes, and notebooks?
  • Highlights — Selected passages in an article, with optional inline notes
  • Article notes — Your thoughts about an article as a whole, visible in the notes panel
  • Notebooks — Freeform notes for synthesis and original writing, optionally linked to source articles
How does search work?

The search bar supports plain text and operators:

  • is:favorite is:unread is:podcast is:book
  • has:summary has:highlights has:notes has:tags
  • tag:philosophy feed:newsletter domain:nytimes.com
  • author:name
  • Combine with OR: tag:ai OR tag:ml
Can I read with text-to-speech?

Yes. Pull Read supports four TTS providers:

  • Apple (built-in) — Works offline, uses macOS system voices
  • ElevenLabs — High-quality neural voices (API key required)
  • OpenAI — Natural-sounding voices (API key required)
  • Kokoro — Open-source local voices, no API key needed

Click the speaker icon on any article to start listening.

AI Features

What AI features are available?
  • Article summaries — One-click summaries of any article
  • Auto-tagging — AI-generated topic tags for your library
  • Daily/Weekly reviews — AI-written digests of your recent reading
Which AI providers are supported?

Pull Read works with multiple providers — bring your own API key:

  • Anthropic (Claude)
  • OpenAI (GPT)
  • Google (Gemini)
  • Apple Intelligence (on supported Macs, no key needed)
  • OpenRouter (access to many models via one key)

Configure your provider and API key in Settings > AI.

Are my articles sent to AI providers?

Only when you explicitly request it — by clicking Summarize, running auto-tagging, or generating a review. Pull Read never sends article content to any server in the background. Your reading habits stay private.

Data & Privacy

Where is my data stored?

Everything is local:

  • Articles — Markdown files in your chosen output folder
  • Highlights, notes, tags — JSON files in ~/.config/pullread/
  • Feed configuration — ~/.config/pullread/feeds.json

Pull Read has no account system and no cloud service. Your data never leaves your machine unless you explicitly use an AI feature.

Can I back up my data?

Yes. Go to Settings > Data and click Export Backup. This creates a single JSON file with all your configuration, highlights, notes, and notebooks. Use Import Backup to restore.

Your articles are already plain Markdown files, so any file backup solution (Time Machine, iCloud, Dropbox) works for those.

Can I sync across devices?

Articles can be synced by putting the output folder in iCloud Drive, Dropbox, or any file sync service. Full annotation sync across devices is on the roadmap.

Obsidian Integration

Can I use Pull Read with Obsidian?

Yes. Since Pull Read saves articles as Markdown with YAML frontmatter, they're natively readable in Obsidian. The simplest setup: point your PullRead output path to a subfolder inside your Obsidian vault (e.g. ~/Obsidian/MyVault/reading/).

How do I set it up?
  1. Open Pull Read Settings
  2. Change your output folder to a subfolder of your Obsidian vault, like ~/Obsidian/MyVault/reading
  3. Your articles will appear in Obsidian's file explorer and be searchable, linkable, and queryable with Dataview
What frontmatter properties are available?

Every article includes these properties that Obsidian and Dataview can use:

---
title: "Article Title"
url: https://example.com/article
author: "Author Name"
domain: example.com
feed: newsletter-name
bookmarked: 2024-01-29T19:05:18Z
lang: en
excerpt: "First 300 characters..."
---

This means you can write Dataview queries like:

```dataview
TABLE author, domain, bookmarked
FROM "reading"
WHERE feed = "tech-newsletter"
SORT bookmarked DESC
```
Can I link to articles from my other notes?

Yes. Since articles live inside your vault, standard Obsidian [[wikilinks]] work. Link to any article from your daily notes, project notes, or anywhere else in your vault.

Do I need an Obsidian plugin?

No. Pull Read's output is standard Markdown with standard YAML frontmatter. No plugin required. Dataview is recommended for querying your reading library but isn't necessary for basic use.

Troubleshooting

The app shows a blank page on launch

On some Intel Macs, the internal server may take a moment to start. Wait a few seconds and click the menu bar icon again. If it persists, quit and relaunch the app. This was improved in v0.3.4.

A feed shows an error in Settings

Feed health monitoring (added in v0.3.4) will show error status for feeds that can't be fetched. Common causes:

  • The feed URL changed or the site went offline
  • The site blocks automated requests (try a different URL)
  • The URL isn't actually a feed (try the site's homepage — Pull Read will auto-discover the feed)

Click the Retry button next to the error to try again.

How do I update Pull Read?

Pull Read checks for updates automatically. When a new version is available, you'll see a notification in the menu bar. Click it to download and install the update. You can also check Releases to see what's new.

Something else isn't working

Check the app logs: open Pull Read, go to Settings, and scroll to the bottom of the About section for the log file path. If you've found a bug, please open an issue on GitHub.
