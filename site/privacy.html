Privacy Policy

Last updated: February 19, 2026

Local-first by design

Pull Read is designed to keep your data on your device. Articles, highlights, notes, tags, and reading history are stored locally on your Mac in the output folder you configure and in ~/.config/pullread/. No data is sent to our servers because we don't operate any.

What stays local

  • All saved articles (markdown files in your output folder)
  • Highlights, notes, and annotations
  • Tags and favorites
  • Reading history and preferences
  • API keys (stored locally in your config directory)
  • TTS audio cache (stored locally in .tts-cache/)
  • Browser TTS and Kokoro local TTS — fully on-device

When data leaves your device

Data is only sent to third parties when you explicitly use certain optional AI features:

  • AI Summaries & Auto-tagging: When you click Summarize or run auto-tagging, article text is sent to your selected LLM provider (Anthropic, OpenAI, Google Gemini, or OpenRouter) using your own API key. Apple Intelligence processes on-device.
  • Weekly Reviews: When scheduled or triggered manually, recent article content is sent to your configured LLM provider.
  • Cloud TTS (OpenAI / ElevenLabs): When you use a paid TTS provider, article text is sent to that provider's API using your dedicated TTS API key. A consent prompt is shown before the first use, and cost estimates are displayed.
  • RSS Feed Fetching: Pull Read fetches RSS/Atom feeds and article URLs you configure. These are standard HTTP requests to the content sources.

Each third-party provider has its own privacy policy governing how they handle data sent through their APIs. We encourage you to review those policies.

API keys

API keys are stored locally in ~/.config/pullread/settings.json. They are never sent to us, never logged, and never shared between features (your Summaries key is separate from your TTS key). Keys are transmitted only to the provider API you selected, over HTTPS.

The viewer server

Pull Read's built-in article reader runs a local HTTP server bound to 127.0.0.1 (localhost only). It is not accessible from other devices on your network.

Analytics and tracking

Pull Read does not include any analytics, telemetry, crash reporting, or tracking. We don't collect any usage data.

Auto-updates

The macOS app checks for updates via a periodic HTTPS request to GitHub Releases to see if a new version is available. No personal data is included in this request.

Children's privacy

Pull Read is not directed at children under 13. We do not knowingly collect information from children.

Changes to this policy

We may update this policy as the product evolves. Changes will be noted with an updated date at the top of this page.
